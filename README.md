# KidsQuiz

## סקירה כללית
KidsQuiz היא מערכת למידה אינטראקטיבית בעברית, הבנויה עם React, Vite ו-Tailwind CSS.
המערכת כוללת עבודה במצב אונליין ובמצב אופליין ל-Windows, עם אותו ממשק משתמש ככל האפשר.

המערכת מיועדת ללמידה, תרגול, מבחנים, ניהול שאלות, עבודה מקומית ללא אינטרנט, ורישוי בסיסי לגרסת הדסקטופ.

## יכולות עיקריות
- עבודה במצב אונליין דרך הדפדפן
- עבודה במצב אופליין דרך אפליקציית Windows מבוססת Electron
- תמיכה ברישיון מקומי לגרסת הדסקטופ
- בחירת שכבת לימוד: כיתה ח׳ או כיתה י״ב
- בחירת נושא לפי שכבת הלימוד
- תמיכה ברמות קושי לפי נושא
- תמיכה בתרגול רגיל ובמבחנים בזמן קצוב
- משוב מיידי על תשובות
- מסך סיכום עם פירוט טעויות והישגים
- מסך ניהול לשאלות ולמשתמשים
- שאלון הולנד ייעודי עם ניתוח טיפוס מוביל ושילוב שני טיפוסים

## מבנה הנתונים של שאלות
השאלות הרגילות במערכת נשמרות בפורמט הבא:

```js
{
  id: 'seed-...',
  grade: 'grade-8' | 'grade-12',
  subject: 'שם הנושא',
  level: 'רמה 1' | 'רמה 2' | 'רמה 3' | null,
  activityType: 'practice' | 'exam',
  type: 'multiple' | 'open',
  text: 'נוסח השאלה',
  options: ['אפשרות 1', 'אפשרות 2'],
  correct_answer: ['תשובה נכונה']
}
```

## שאלון הולנד
שאלון הולנד משולב במערכת כנושא ייעודי נפרד, ואינו משתמש במנגנון נכון/לא נכון של החידון הרגיל.

### מה כולל השאלון
- 24 היגדים
- דירוג אישי לכל היגד בין 0 ל-3
- חישוב ניקוד לפי מודל RIASEC
- זיהוי טיפוס מוביל
- זיהוי שילוב של שני טיפוסים מובילים
- תצוגת התפלגות ניקוד לפי סוגים

### מה לא מוצג כרגע
- אין במערכת המלצות מקצוע או כיווני קריירה לפי קוד הולנד

## גרסת אופליין ל-Windows
גרסת האופליין מבוססת Electron, ומשתמשת בפרונט הקיים של המערכת.

### מאפיינים
- אותו UI בסיסי כמו גרסת הדפדפן
- הרצה מקומית ללא אינטרנט
- רישיון מקומי
- נתונים ותוצאות נשמרים מקומית
- אפשרות בנייה ל-EXE ולקובץ התקנה

### קבצים חשובים
- `electron/main.cjs`
- `electron/preload.cjs`
- `scripts/generate-license.mjs`
- `docs/OFFLINE_WINDOWS.md`

## הורדה של גרסת האופליין
המערכת נשענת על משתנה סביבה בשם:

```env
VITE_OFFLINE_DOWNLOAD_URL
```

המשתנה צריך להצביע לקובץ התקנה קבוע בשם:

```text
KidsQuizOfflineSetup.exe
```

בפועל אפשר לארח את קובץ ההתקנה דרך GitHub Releases ולשמור את אותו שם קובץ בין גרסאות.

## רישוי לגרסת הדסקטופ
מנגנון הרישוי הקיים מאפשר:
- טעינת קובץ רישיון מקומי
- מספר רישיון ייחודי
- התחלת ספירה לאחור של 180 ימים מהפעלה ראשונה
- המשך ספירה גם אם מעבירים את הקובץ למחשב אחר
- חסימת יצירת משתמש חדש באופליין

יצירת רישיון לדוגמה:

```powershell
npm run license:generate -- --customerName="Test User" --customerId="TEST-001" --durationDays=180 --out="release\licenses\test-license.json"
```

## אחסון מקומי
המערכת משתמשת ב-LocalStorage לשמירת:
- משתמש נוכחי
- מאגר שאלות
- מצב עבודה
- נתוני משתמשים ותוצאות מדומות עבור מצב אונליין מקומי
- תוצאות אופליין מקומיות

## הערות תחזוקה
- כל הטקסטים בממשק נשמרים בעברית
- הכיוון הכללי של המערכת הוא RTL
- יש להעדיף הרחבה של שכבת הנתונים המרכזית במקום קידוד קשיח במסכים
- שאלון הולנד הוא flow עצמאי, ולכן לא נכון להכניס אותו לתוך מנגנון ה-quiz הסטנדרטי

## הערות לעבודה עתידית
- שיפור ודיוק של מאגר השאלות לכל נושא
- הרחבת שאלון הולנד אם יידרש
- שדרוג מנגנון הרישוי באופליין
- חיבור עתידי ל-backend אמיתי במקום הדמיה מקומית
